## 📕 背景

ユーザーからの情報提供と公式情報の確認により，2025-12-07 開催時点の README が前提とする手順の一部が成立しなくなったことが判明した．主な変化は次の 3 点である．

- R2P2 の最新版を Raspberry Pi Pico にインストールすると，ストレージとしてマウントされなくなった．
- hasumikin 氏により Web Terminal が開発され，TeraTerm 等のターミナルソフトのインストールが不要になった．
- `app.rb` 等のファイル転送は Web Terminal の Upload／Download を介して行えるようになった．

本改修では，旧版を Git tag として保全しつつ，main を最新挙動に合わせて改める．

## 📕 確認済みの事実

公式情報により確認できた事実を以下に列挙する．本文記述の根拠とする．

- R2P2 は `picoruby/picoruby` の `mrbgems/picoruby-r2p2` に統合された．旧 `picoruby/R2P2` リポジトリは consolidate 済みと案内されている．
- 最新リリースは `3.4.2`（2026-03-29 公開）である．UF2 アセットは `R2P2-PICORUBY-3.4.2-PICO-*.uf2`（RP2040 用）と `R2P2-PICORUBY-3.4.2-PICO2_W-*.uf2`（Pico 2 W 用）が提供される．
- 公式 setup ガイドは「UF2 ファイルがコピーされたあと Pico は自動再起動し，`RPI-RP2`（または `RP2350`）ドライブはファイルマネージャから消える」と明記する．
- Web Terminal は hasumikin 氏の制作で，`https://picoruby.org/terminal` にて提供される．Web Serial API を用いるため Chrome／Edge（デスクトップ 89+）が必要であり，R2P2 3.4 以降が前提となる．
- Web Terminal は File Editor を備え，`Plain`／`Compile`／`Download`／`Upload` ボタンと `Open local file` 機能を持つ．
- 公式 terminal-emulator ガイドは Web Terminal を第一選択とし，TeraTerm／screen／minicom はフォールバック扱いと位置づける．

## 📕 検証結果（2026-05-22）

残った裏取り 4 項目をサブエージェント並列調査とユーザー実機確認で解消した．以下に要点を整理し，Phase C 以降に持ち込む追加方針を示す．

### 📗 V-1：`/home/app.rb` の自動実行は健在

`picoruby/picoruby` master の `mrbgems/picoruby-shell/shell_executables/r2p2.rb` 13〜17 行で `File.exist?("#{ENV['HOME']}/app.rb")` を判定し `load` する．`ENV['HOME']` は `/home` で固定である．優先順位は `app.mrb` > `app.rb` > DFU ブートマネージャ．起動時 2 秒以内に `s` キー押下で自動実行を抑止できる．したがって教材が前提とする `/home/app.rb` の挙動は維持されている．

### 📗 V-2：API 互換性と対象 R2P2 版

公式 UF2 リリースは現状 `3.4.2` のみであり，`3.4.3`／`3.4.4`／`3.4.5` はタグのみで配布物がない．Web Terminal の WASM が `microruby 3.4.5` と表示されるのは `@picoruby/wasm-wasi` パッケージ独自のスナップショットで，ファームウェアとは別系統である．3.4.2 は mruby/c VM の最終版で，次バージョンは mruby VM へ切り替わる旨がリリースノートに明記される．**対象は 3.4.2 固定で確定**とする．

教材で使う ADC・UART・GPIO・PWM の API には 0.4.x から 3.4.2 まで破壊的変更が見当たらない．`app.rb`，README の irb チュートリアル，応用課題コードはコード修正なしで 3.4.2 でも動作する見込みである．ランタイム挙動の最終確認は Phase E に残す．

なお上流 `mrbgems/picoruby-pwm/README.md` は `duty(0.0-1.0)` と記述するが C 実装は `0-100` を期待する．本教材の `duty(50)` 等の表記は実装と整合しており，このまま維持してよい．

### 📗 V-3：RP2040 × Web Terminal は教材範囲をカバー

`mrbgems/picoruby-r2p2/ports/rp2040` が RP2040 を主軸に据えており，シリアル接続・`irb`／`ls`／`cat`・File Editor の Upload／Download は無印 Pico でフル動作する見込みである．RP2040 固有の重大バグは見当たらない．[picoruby/picoruby#415](https://github.com/picoruby/picoruby/issues/415) 等の Issue も板由来ではない．

Phase C で次の注意を README に追記する．

- ブラウザは Chrome／Edge（デスクトップ 89+）必須である．
- UF2 は `R2P2-PICORUBY-3.4.2-PICO-*.uf2`（無印 Pico 用）を選ぶ．Pico 2 W 用と取り違えない．
- 初回起動が不安定なら [`flash_nuke.uf2`](https://picoruby.org/rom-corruption) でフラッシュを初期化してから書き直す．
- Web Terminal のシェルへ複数行を一括コピペすると失敗する．長いコードは File Editor の Upload で配置する．
- 起動時 2 秒以内の `s` キー押下で `/home/app.rb` の自動実行を一時的に抑止できる．

「Select closest version number to your R2P2」ドロップダウンの正確な仕様は公式に明記されておらず，バイトコード互換の救済手段と推定される．3.4.2 を焼いた Pico は `Latest` のままで問題ない見込みである．

### 📗 V-4：Web Terminal の Upload／Download はフルパス指定

ユーザー実機確認により仕様確定．File Editor 直上のテキストフィールドに `/home/app.rb` のようなフルパスを入力し `Upload` を押下する．Download も同じ欄にパスを入れて取得する．Phase C-4 はこの形式に合わせて手順を書く．
